# Reference: Report Layout

The final report should be clear enough for a business owner to read without opening the raw Amazon files.

## Page or section 1: Income and expenses

Include:

- report title
- marketplace
- reporting month
- income table
- expense table
- gross income
- short variance note, if needed

## Page or section 2: Product performance

Include:

- clean product name
- units sold
- B2B units, if available
- product revenue
- B2B revenue, if available

## Formatting principles

- Keep labels consistent month to month.
- Keep the report compact.
- Use the same section order every month.
- Do not hide variance or mismatches.
- Use clear notes instead of unexplained numbers.

## Output package

A complete monthly output package should include:

```text
00_RAW_DOWNLOADS/
01_EXTRACTED_DATA/
02_REPORT_BUILD/
03_QA_CHECKS/
04_FINAL_REPORT/
05_LOGS/
```

The final report is only one part of the output. The traceable folder is what makes the workflow trustworthy.
